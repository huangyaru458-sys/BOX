\# 物联网系统开发实训（完整官方实训要求文档） ## 一、实训目的 本实训旨在帮助学生建立物联网系统开发的完整知识体系与实践能力。通过本课程的学习与实践，学生应达成以下目标： 1. 了解物联网的概念、特点和应用领域，理解物联网的基本概念，掌握物联网的定义、核心三层架构（感知层、网络层、应用层）及其在典型行业中的应用模式。 2. 掌握嵌入式硬件开发基础：熟悉ESP32芯片的硬件架构、外设接口（GPIO、ADC、I2C、SPI、UART等）及其功能特性，能够根据应用需求选择合适的硬件方案。 3. 熟练使用Arduino IDE开发环境：掌握Arduino框架下ESP32工程的创建、编译、烧录与调试流程，能够独立完成嵌入式程序的开发与调试。 4. 掌握物联网后端平台构建方法：学习使用Node-RED可视化编程工具搭建物联网数据接收、处理与存储平台，理解低代码开发在物联网领域的应用。 5. 掌握多种通信协议的应用：深入理解WiFi、蓝牙（经典蓝牙/SPP与BLE）、HTTP、MQTT、WebSocket等协议的工作原理与适用场景，能够根据实际需求选择合适的通信方案实现数据高效传输。 6. 具备跨平台移动端开发能力：学习使用HBuilderX开发基于HTML5+的移动端应用，掌握uni-app或HTML5+ App的开发流程，实现物联网系统的移动端监控与控制。 7. 具备全栈物联网系统开发能力：能够独立完成从传感器数据采集、无线传输、后端数据处理、数据库存储到Web端与移动端可视化的完整系统设计与实现。 ## 二、实训内容 ### 2.1 项目概述 学生需综合运用本门课程所学物联网系统开发技术，自行拟定一个具有实际应用价值的物联网场景，独立完成从需求分析到系统实现与测试的完整开发流程。最终交付物为一个功能完整的物联网系统原型（含Web端与移动端）及相应的设计文档。 ### 2.2 系统功能要求 设计并实现的物联网系统应涵盖以下核心功能模块： #### （一）数据采集层（感知层） 1. 使用ESP32微控制器连接各类传感器（如温湿度传感器、空气质量传感器、人体红外传感器、超声波测距模块、姿态传感器等），实现物理世界数据的实时采集。 2. ESP32对采集的原始数据进行必要的预处理（滤波、格式转换、数据校验等），过滤无效脏数据。 #### （二）数据传输（网络层） 1. ESP32通过WiFi无线网络将采集到的传感器数据发送至后端服务器。 2. 采用MQTT协议作为ESP32与后端之间的主要通信协议，合理设计Topic主题结构，根据数据特性选择合适的QoS等级。 #### （三）数据存储层 1. 后端平台接收并解析ESP32上报的数据，将其持久化存储至数据库中；推荐使用SQLite轻量级数据库，也可选用MySQL、MongoDB等数据库系统。 2. 设计合理的数据表结构，支持历史数据的查询、统计与分析；配置定时清理任务，实现数据保留策略（默认保留近30天数据）。 #### （四）后端服务层（应用层） 1. 使用Node-RED作为物联网后端开发平台，通过MQTT In节点接收设备上报数据。 2. 利用JSON节点、Function节点实现数据的校验、转换与业务逻辑处理。 3. 设计并实现RESTful HTTP API接口，为前端应用（Web端、移动端）提供标准化的数据访问与设备控制通道，接口包含：    - GET /api/data/latest — 获取设备最新实时数据    - GET /api/data/history?device_id=xxx&limit=50 — 获取设备历史时序数据    - POST /api/control — 下发设备远程控制指令    - GET /api/devices — 获取全部设备列表与在线状态 4. 实现设备在线状态监控、异常告警推送功能。 5. 配置CORS跨域响应头，允许移动端、Web端跨域访问API（生产环境限制指定域名，实训环境放开全部域名）。 6. 通过httpstatic节点托管Web端静态页面（HTML/CSS/JS）。 #### （五）Web前端展示与控制层（Vue/原生HTML5） 1. 采用Vue框架或原生HTML5 + CSS + JavaScript构建响应式Web用户界面。 2. 前端通过fetch() API封装HTTP请求函数，统一调用后端RESTful接口。 3. 使用Chart.js实现数据可视化仪表盘、趋势图、历史曲线图表。 4. 支持反向控制：用户可通过Web界面的按钮、开关等控件发送控制指令。 5. 通过定时轮询（setInterval）或WebSocket实现设备在线/离线状态实时检测。 6. 通过CSS Flexbox、Grid与Media Query实现响应式布局，适配PC端、平板浏览器访问。 #### （六）移动端应用层（HBuilderX uni-app开发） 使用HBuilderX开发一款跨平台移动端App（基于HTML5+或uni-app框架），实现以下功能： 1. 实时数据查看：通过HTTP请求调用后端API，以卡片、仪表盘形式展示温湿度、空气质量、箱体容量等传感器实时数据。 2. 历史数据图表：使用移动端图表库（uCharts/ECharts）展示24小时环境趋势曲线，支持滑动查看历史记录。 3. 远程设备控制：提供开关、按钮交互控件，通过HTTP POST请求下发控制指令至后端，远程操控ESP32外接执行器。 4. 设备状态监控：实时显示设备在线/离线状态，设备离线时给出视觉遮罩、文字提示。 ### 2.3 系统标准三层架构（含移动端） 完整三层物联网分层架构： 1. **前端展示层**    - Web端：原生HTML5+CSS+JS/Vue，包含数据卡片、趋势图表、远程控制面板、设备在线状态、告警弹窗模块；    - 移动端：HBuilderX uni-app，包含实时数据卡片、历史图表、告警推送、远程控制页面。    - 通信方式：HTTP / WebSocket 对接Node-RED后端API服务。 2. **后端服务层（Node-RED平台）**    - MQTT接入节点：接收ESP32上报遥测、告警消息；    - 数据预处理节点：JSON解析、数值类型校正、脏数据过滤；    - 业务逻辑节点：告警判断、设备在线心跳维护；    - HTTP REST API节点：对外提供数据查询、设备控制接口；    - 数据库交互节点：SQLite/MySQL持久化存储时序数据；    - 静态Web托管节点：存放前端网页资源。 3. **感知控制层（ESP32终端）**    - 传感器驱动模块：温湿度、气体、超声波、人体红外、MPU6050姿态采集；    - 执行器驱动模块：LED指示灯、蜂鸣器、继电器风扇、矩阵键盘；    - 数据采集预处理模块：滤波、数值换算；    - WiFi连接管理模块：STA联网/AP离线双模式、断线自动重连；    - MQTT客户端模块：定时发布采集数据、订阅远程控制指令；    - 本地WebServer模块：断网AP模式下提供局域网本地HTTP接口。 ### 2.4 各端开发技术要点 #### 硬件端（ESP32 Arduino IDE）开发要点 1. 根据传感器接口类型（GPIO/ADC/I2C）完成硬件接线，严格遵循共地、隔离供电规范。 2. 使用Arduino框架编写ESP32程序，采用FreeRTOS多任务开发，禁止全局阻塞delay()，避免任务死锁、网络丢包。 3. 实现WiFi连接管理，包含断线自动重连、STA/AP双模式自动切换机制；使用Preferences库将WiFi账号密码存入NVS，实现断电记忆配网。 4. 实现MQTT客户端，按照预设Topic主题定时发布传感器上报数据，订阅控制主题接收并执行反向控制指令。 5. I2C多设备并联时增加互斥同步机制，防止多任务抢占总线通信死锁。 6. 可选拓展功能：BLE蓝牙辅助配网，简化设备首次入网操作。 #### 后端Node-RED开发要点 1. 在Node-RED中配置本地/公共MQTT Broker，订阅设备遥测、告警主题，监听控制下发主题。 2. 使用JSON节点、Function节点完成报文解析、数据类型强制转换，过滤超出量程的无效采集数据。 3. 设计传感器时序数据表，通过SQLite/MySQL节点实现数据持久化；配置定时清理节点，自动删除30天前历史数据。 4. 规范开发4类RESTful API接口，区分实时数据、历史数据、设备控制、在线设备列表接口。 5. 配置Catch异常捕获节点、Status状态节点，实现程序异常捕获、设备心跳在线监控。 6. 配置CORS跨域响应头，实训环境允许所有前端域名跨域调用接口。 #### Web前端（Vue/原生HTML5）开发要点 1. 创建标准HTML5文档结构，使用语义化标签，统一CSS变量管理界面主题色彩。 2. 封装通用fetch()请求函数，统一处理HTTP请求、异常捕获、返回码校验。 3. 集成Chart.js图表库，实现双轴折线图展示温湿度、空气质量24小时变化趋势。 4. 使用setInterval定时轮询后端状态接口，实时刷新设备在线状态；可选WebSocket方案实现服务端主动推送。 5. 搭建远程控制面板，下发开门、启停风扇等控制指令；实现异常告警弹窗提示。 6. 使用Flex+Grid布局、Media Query媒体查询，完成PC、平板自适应响应式页面。 #### 移动端（HBuilderX uni-app）开发要点 1. 搭建多页面工程：首页实时数据、历史图表页、远程控制页、离线本地模式切换页。 2. 封装uni.request通用请求工具，统一处理网络异常、离线模式地址切换。 3. 集成uCharts图表插件渲染历史趋势曲线，页面加载完成后再渲染图表，避免空白画布死锁。 4. 提供离线本地模式切换逻辑，切换后请求地址改为ESP32本地热点网关`192.168.4.1`。 5. 微信开发者工具调试时，支持关闭合法域名校验，适配本地局域网IP接口调试。 6. 设备离线时添加页面遮罩、文字提示，异常告警弹窗提醒运维人员。 ## 三、实训交付文档硬性要求 完成系统设计后，需提交完整实训设计文档，文档必须包含以下完整章节： 1. **绪论**：包含清晰明确的项目目标陈述，阐述系统应用背景、行业痛点、项目开发意义与实训学习目标。 2. **需求分析**：准确描述系统应用领域、目标用户；清晰区分系统功能需求（感知层、后端、Web前端、移动端）与非功能需求（实时性、稳定性、容错性、兼容性）。 3. **系统设计与架构**：详细阐述三层物联网整体分层架构，绘制系统架构图，说明感知层、后端、前端各组件之间的数据交互关系；包含完整硬件接线规范、引脚分配表、通信接口定义。 4. **系统实现**：清晰描述硬件、后端、Web、移动端各模块开发方法、开发步骤，提供硬件接线图、通信数据流流程图、接口完整规范。 5. **系统测试与评估**：说明硬件单元测试、通信联调测试、全系统集成测试的测试方法与测试策略；对系统运行性能、并发可靠性、容错能力进行客观评估，分析系统不足与优化改进方向。 ## 四、实训系统整体运行逻辑总结 本实训系统分为硬件感知终端、Node-RED后端服务、Web+PC/移动端APP三大部分： 1. ESP32与各类传感器、执行器硬件配合，完成环境、箱体状态数据采集，本地预处理后通过WiFi+MQTT无线网络发送至Node-RED后端平台； 2. 后端Node-RED接收并解析MQTT设备报文，完成数据清洗校验后存入数据库持久化；同时封装标准化HTTP API接口，提供前端数据查询、远程控制通道； 3. Web前端（Vue）、移动端（HBuilderX uni-app）通过HTTP/WebSocket调用后端API，实时获取传感器数据、设备运行状态，同时可下发反向控制指令至后端，经由MQTT下发至ESP32终端完成设备远程操控； 4. 额外支持断网离线应急模式：WiFi断开后ESP32自动开启AP热点，手机直连热点调用ESP32内置本地HTTP接口，绕开后端服务器独立完成本地数据查看、管理员密码开锁操作。
